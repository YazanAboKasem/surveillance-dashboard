# دليل إضافة جهاز Jetson جديد إلى نظام المراقبة (RoadShield)

هذا التقرير يوضح الخطوات البرمجية والتقنية اللازمة لإضافة جهاز Jetson جديد (نقطة مراقبة جديدة) إلى لوحة التحكم (Laravel Dashboard). تم تصميم النظام ليدعم حتى 20 جهازاً أو أكثر بشكل مرن ومستقل.

---

## أولاً: الإعداد في جانب لوحة التحكم (Laravel Dashboard)

### 1. تعديل ملف الإعدادات `config/surveillance.php`
يتعين عليك إضافة كتلة إعدادات جديدة داخل مصفوفة `devices`.
افتح الملف [surveillance.php](file:///home/yazan/projects/rs-python-stream/surveillance-dashboard/config/surveillance.php) وقم بإضافة الكود التالي كعنصر جديد:

```php
[
    'id'              => 'jetson-2', // معرف فريد للجهاز الجديد (يستخدم برمجياً وفي مسارات الـ API)
    'name'            => 'Jetson Orin — Back Gate', // الاسم الظاهر للجهاز في لوحة التحكم
    'location'        => 'Back Yard / Loading Area', // الموقع الجغرافي للجهاز
    'host'            => env('JETSON2_HOST', '127.0.0.1'), // عنوان الـ IP المحلي أو النطاق
    'hls_port'        => env('JETSON2_HLS_PORT', 8888), // منفذ HLS لـ MediaMTX الخاص بالجهاز
    'webrtc_port'     => env('JETSON2_WEBRTC_PORT', 8889), // منفذ WebRTC لـ MediaMTX
    'hls_base_url'    => env('JETSON2_HLS_URL'), // مسار Cloudflare Tunnel إذا كان مستخدماً (اختياري)
    'webrtc_base_url' => env('JETSON2_WEBRTC_URL'), // مسار نفق WebRTC (اختياري)
    'tunnel_cache_key' => 'tunnel_jetson_2', // مفتاح الكاش لتسجيل نفق Cloudflare Tunnel تلقائياً
    'api_token'       => env('JETSON2_TOKEN', env('SURVEILLANCE_TOKEN')), // رمز الحماية الخاص بالجهاز
    'enabled'         => true, // تفعيل أو تعطيل الجهاز
    'cameras' => [
        [
            'id'         => 'cam4', // معرف فريد للكاميرا
            'label'      => 'Camera 4 — Cargo Entry', // اسم الكاميرا الظاهر
            'path'       => 'cam4', // مسار البث الرئيسي HD في MediaMTX
            'path_sub'   => 'cam4_sub', // مسار البث الفرعي SD
            'path_ultra' => 'cam4_ultra', // مسار البث منخفض الجودة
            'path_live'  => 'cam4_live',
            'ip'         => '192.168.1.70', // عنوان IP الكاميرا الشبكي (للـ PTZ)
            'enabled'    => true,
            'ptz'        => true, // تفعيل ميزة تحريك الكاميرا إذا كانت تدعمها
        ],
        // يمكنك إضافة كاميرات إضافية للجهاز هنا (مثلاً كاميرا 5 وكاميرا 6)
    ],
],
```

### 2. تحديث ملف البيئة `.env`
أضف المتغيرات الخاصة بالجهاز الجديد في ملف `.env` الخاص بـ Laravel لربط المنافذ والرموز السرية ديناميكياً:
```env
JETSON2_HOST=192.168.1.150
JETSON2_HLS_PORT=8888
JETSON2_WEBRTC_PORT=8889
JETSON2_TOKEN=your_secure_token_for_jetson_2
```

## ثانياً: الإعداد في جانب جهاز Jetson الجديد (Client-Side)

لكي يتصل جهاز Jetson الجديد بلوحة التحكم ويبدأ ببث الفيديو وإرسال البيانات، يجب إعداد البرمجيات التالية عليه:

### 1. إعداد الـ API Token
في ملف الإعدادات أو ملف البيئة `.env` الخاص بجهاز Jetson الجديد، قم بضبط رمز المصادقة ليتطابق مع الرمز المحدد له في لوحة التحكم:
```env
SURVEILLANCE_TOKEN=your_secure_token_for_jetson_2
```

### 2. ضبط سكربت ربط النفق `connect-to-server.sh`
إذا كنت تستخدم **Cloudflare Tunnel** للوصول للبث من الخارج:
* قم بضبط عنوان تسجيل النفق ليرسل الطلب إلى لوحة التحكم مع تمرير معرف الجهاز الجديد في الطلب (أو تحديث سكربت الإرسال ليتناسب مع كاش المفتاح الجديد `tunnel_jetson_2`).
* يقوم السكربت بإرسال طلب `POST` إلى عنوان سيرفر Laravel لتسجيل رابط النفق ديناميكياً في الكاش.

### 3. ضبط برنامج البث `MediaMTX`
* تأكد من إعداد ملف `mediamtx.yml` على الجهاز الجديد ليحتوي على المسارات المعرفة في لوحة التحكم (مثل `cam4`, `cam4_sub`, `cam4_ultra`).
* تأكد من أن الكاميرات تبث للمسارات الصحيحة على خادم MediaMTX المحلي.

### 4. تشغيل برامج التحكم والتشخيص
قم بتشغيل السكربتات التالية على الـ Jetson الجديد:
* `camera-control.py`: للتحكم بميزات PTZ (التحريك والزووم) عبر بروتوكول ISAPI.
* `jetson_ws_client.py`: للاتصال بخادم الـ WebSockets الخاص باللارافل لتمكين التشخيص الفوري والاستجابة اللحظية للأوامر (مثل إعادة التشغيل والتنظيف وسحب حالة الاتصال).

## ثالثاً: التحقق من التفعيل

بعد إجراء الخطوات السابقة:
1. افتح لوحة التحكم واذهب إلى صفحة **Devices** (الأجهزة) للتأكد من ظهور الجهاز الجديد وحالته (متصل/غير متصل).
2. ادخل إلى **Monitoring Room** (غرفة المراقبة) لتجد أن كاميرات الجهاز الجديد تم تجميعها تلقائياً داخل إطار خاص يحمل اسم الجهاز الجديد وبثوثه الحية.
3. اضغط على زر **Settings** الخاص بالجهاز للتأكد من عمل الفيتشرات الخاصة به (مثل فحص التشخيص Diagnostic وتعديل جودة البث).
